## 1. Technical Overview
Bibliotheca Modern is a premium digital bookstore platform utilizing a **Next.js 15 (App Router)** stack with a **Server-First** logic.

### Core Stack
- **Framework**: Next.js 15.1.6
- **Language**: TypeScript
- **Database**: SQLite (better-sqlite3)
- **Styling**: Tailwind CSS 4.0
- **Animations**: GSAP & Framer Motion
- **E-book Engine**: Epub.js

---

## 2. Design Language
### Visual Philosophy
Modern Minimalist / Editorial. Focus on large typography, whitespace, and high-contrast dark themes.
- **Background**: `#0a191f`
- **Accent**: `#00e5ff` (Cyan)
- **Typography**: `Playfair Display` (Serif) & `Inter` (Sans-Serif)

---

## 3. System Architecture
Uses a **Unidirectional Data Stream** where Server Components (RSC) fetch data directly from SQLite, minimizing client JS and maximizing SEO.

### Data Model
- **`books`**: Metadata and binary links.
- **`profiles`**: User account and metrics.
- **`user_library`**: Junction table for ownership and reading status.
- **`admin_users`**: Administrative credentials.

---

## 4. Deployment Strategy
Containerized via **Docker** and proxied by **Nginx**.
- **Volume Persistence**: `./data` mapped to `/app/data` for SQLite and asset storage.
- **Build**: Multistage Dockerfile for Next.js production execution.

---

## 5. Frontend Layer
- **Client Components**: Only for animations, state-heavy filters, and the **E-book Reader**.
- **State management**: 
  - **Server-driven**: Route rendering.
  - **Client-driven**: `CartContext` with `localStorage` persistence.
- **Icons**: Lucide React.

---

## 6. Backend Layer
- **Runtime**: Node.js 20+.
- **Database Connection**: `better-sqlite3` enabled in **WAL (Write-Ahead Logging)** mode.
- **Security**: 
  - **Bcrypt** password hashing.
  - **HttpOnly** cookie-based sessions for users and admins.

---

## 7. API & Website Working
Instead of REST/GraphQL, the project uses **React Server Actions** (`'use server'`).
- **`getBooks`**: Modular SQL builder for catalog filtering.
- **`uploadBook`**: Administrative multipart form processing for metadata and e-book binaries.
- **Search/Sort**: URL-synchronized query parameters drive server-side re-execution of data fetches.

---

## 8. Common Questions (FAQs)
- **No books showing?**: Run `node scripts/seed-db.js`.
- **E-book not loading?**: Verify the `/ebooks/` path and binary exists in `public/`.
- **Admin Password Reset**: Update the `admin_users` table with a new Bcrypt hash via a SQLite client.
- **Checkout Process**: Currently records purchases in the library but requires Stripe/Braintree integration for payments.
